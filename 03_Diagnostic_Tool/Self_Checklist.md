# ✅ 소버린 AI 등급 판별 문진표 (Diagnostic Checklist)
> **Self-Assessment Flow for Sovereign AI T-Class 2.0**

귀사의 모델이 어느 주권 등급에 해당하는지 명확히 판별하기 위한 **자가 진단 가이드**입니다.
순서대로 질문에 답하며 화살표를 따라가십시오.

---

### [Step 1] 소버린 AI 진입 여부 (T3 vs T4)
> **Q1. 모델의 가중치(Weights)를 처음부터 자체 학습(Pre-training from Scratch) 시켰습니까?**
> *(기존 모델의 가중치를 이어서 학습하거나, 섞지 않았습니까?)*

* **No (아니요)** → 기존 가중치를 활용했습니다.
    * ▶ **[T2 또는 T3]** (비 소버린 구간)
    * *진단 종료: 귀사의 모델은 효율적이지만, 기술 주권은 원작자에게 종속되어 있습니다.*
* **Yes (네)** → 처음부터 학습했습니다.
    * ▶ **[Step 2]**로 이동하십시오.

---

### [Step 2] 아키텍처 독창성 판별 (T4 vs T5)
> **Q2. 모델 구동을 위해 `trust_remote_code=True` 옵션이 필요하며, HuggingFace 표준 클래스와 호환되지 않는 독자적인 코드를 사용합니까?**
> *(모델링 코드, 토폴로지, 연산 그래프가 독자적입니까?)*

* **No (아니요)** → 표준 아키텍처와 호환됩니다. (Llama, Mistral 등)
    * ▶ **[Step 3]**로 이동 (T4 세부 등급 판별)
* **Yes (네)** → 독자적인 모델링 코드를 짰습니다.
    * ▶ **[Step 4]**로 이동 (T5 진위 여부 확인)

---

### [Step 3] T4 세부 등급 판별 (T4-1 vs T4-2)
> **Q3. 원본 아키텍처의 `config.json` (레이어 수, 히든 사이즈 등)을 변경하거나 구조적 최적화를 수행했습니까?**

* **No (아니요)** → 원본 설정 그대로 학습만 했습니다.
    * 🏁 **판정: [T4-1. 아키텍처 어답터 (Architecture Adopter)]**
    * *의의: 데이터 주권은 확보했으나, 기술적 독창성은 낮음.*

* **Yes (네)** → 사이즈를 키우거나(Scaling), 구조를 일부 변경(Modification)했습니다.
    * 🏁 **판정: [T4-2. 아키텍처 스케일러 (Architecture Scaler)]**
    * *의의: 글로벌 표준을 가장 잘 활용하는 실용적 소버린 모델.*

---

### [Step 4] 기술 리더십 최종 판별 (T5 vs T6)
> **Q4. 이 모델이 국산 AI 반도체(NPU) 위에서 학습/구동되며, 국산 클라우드 인프라에 안착되어 있습니까?**

* **No (아니요)** → 외산 GPU/클라우드를 사용합니다.
    * 🏁 **판정: [T5. 네이티브 아키텍처 (Native Architecture)]**
    * *조건: 한국어 전용 토크나이저를 원천 구축했어야 함.*

* **Yes (네)** → 하드웨어와 인프라까지 국산입니다.
    * 🏁 **판정: [T6. 풀스택 소버린 (Full-Stack Sovereign)]**
    * *의의: 국가 안보 수준의 완전한 기술 독립.*

---

## 📝 빠른 요약표 (Quick Summary)

| 질문 | 답변 | 판정 등급 |
| :--- | :--- | :--- |
| **자체 학습(Pre-train) 아님** | No | **T0 ~ T3** (비 소버린) |
| **자체 학습** + 표준 코드 그대로 | Yes + No | **T4-1** (어답터) |
| **자체 학습** + 구조/파라미터 변경 | Yes + Modified | **T4-2** (스케일러) |
| **자체 학습** + 독자 코드/토크나이저 | Yes + Custom Code | **T5** (네이티브) |
| **T5 모델** + 국산 NPU/Cloud | Yes + Sovereign Infra | **T6** (풀스택) |
